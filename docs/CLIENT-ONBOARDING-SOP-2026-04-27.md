Client Onboarding SOP  
InfraFlow ERP

1. Objective
This SOP defines the standard process for onboarding a new client company onto InfraFlow ERP in a safe and professional way.

2. Pre-Onboarding Inputs Required
Collect:
- company legal name
- branch / site name
- owner full name
- owner mobile number
- owner email if used
- GST / tax details
- address
- operational module requirements
- approximate number of plants
- approximate number of vehicles
- approximate number of users
- pricing package confirmation

3. Internal Readiness Check
Before onboarding:
- production system is healthy
- credentials are ready
- support person is assigned
- commercial approval is complete
- scope is documented

4. Create Client Company
Perform:
- create company record
- assign company code
- create owner login
- verify module access
- verify company is active

5. Secure Credential Handover
Share through approved secure channel only:
- client portal URL
- company code
- username
- temporary password

Rules:
- do not share credentials in open groups
- if possible, send username and password separately
- instruct owner to change password immediately after first login

6. Minimum Setup Checklist
Configure or verify:
- company profile
- branches if applicable
- plants
- vehicles
- parties
- materials
- shifts
- commercial rates
- employee records if included in initial phase

7. First Login Validation
Confirm with client:
- company code opens correct login context
- login succeeds
- password change completed
- dashboard opens
- one or two core screens load correctly

8. Initial Training Flow
Train in this order:
- login and password hygiene
- dashboard understanding
- masters maintenance
- dispatch workflow
- reports review
- role/access expectations

9. Go-Live Validation
Confirm:
- owner can log in
- daily operator/manager can log in if applicable
- one real or sample dispatch flow works
- one report flow works
- support contact is known

10. Post-Go-Live Monitoring
Monitor for first few business days:
- login issues
- missing master data
- report confusion
- workflow blocking bugs
- owner feedback

11. Handover Completion
Mark onboarding complete only when:
- credentials handed over securely
- password changed
- company profile verified
- required masters loaded
- first workflow validated
- owner trained on basics

12. Internal Completion Record
Record:
- company name
- company code
- onboarding date
- onboarding owner
- package sold
- modules enabled
- pending actions
- support follow-up date
